# RCA Framework

Run a Root Cause Analysis on the supplied issue using 5 Whys and a simple cause tree.

Required output:

- Problem statement
- Observed symptoms
- 5 Whys chain
- Confirmed root cause
- Immediate mitigation
- Long-term prevention
- Validation step

Rules:

- Separate facts from hypotheses.
- Prefer concrete evidence over generic explanations.
- Do not invent metrics, process details, or owners.
- Mark missing inputs with `[MISSING]`.
